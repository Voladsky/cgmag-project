# Техническое описание проекта: «Динамическая водная поверхность с кораблями»

## 1. Общие сведения

Проект представляет собой демонстрационное приложение на C++/OpenGL, визуализирующее анимированную водную поверхность с эффектами окружения (skybox), отражениями, пеной и движущимися кораблями. В основе генерации волн лежит сумма направленных волн Герстнера, вычисляемая в геометрическом шейдере. Реализована интерактивная камера (FPS‑стиль), переключение двух наборов текстур для неба (клавиша **B**) и выбор режима коэффициента Френеля (клавиша **F**) для управления интенсивностью отражений.

Проект написан на **C++11** с использованием библиотек:

- **GLFW** – создание окна и обработка ввода;
- **GLAD** – загрузка расширений OpenGL 3.3 Core;
- **GLM** – математические операции (матрицы, векторы, кватернионы);
- **Assimp** – импорт 3D‑модели корабля в формате OBJ;
- **stb_image** – загрузка текстур (PNG, JPG).

Сборка выполняется с помощью **Makefile** (компилятор **g++**).

## 3. Зависимости и сборка

### 3.1 Требования к системе

- Операционная система Linux (или другая с поддержкой X11/Wayland + OpenGL);
- Компилятор с поддержкой C++11;
- Установленные библиотеки: `libglfw3-dev`, `libgl1-mesa-dev`, `libassimp-dev`, `libstb-dev` (или `stb_image.h` должен быть доступен в системе).

### 3.2 Makefile

```makefile
CXX = g++
CXXFLAGS = -std=c++11 -Wall -O2 -I.
LDFLAGS = -lglfw -lGL -lassimp -ldl

SRC = main.cpp glad.c
OUT = water_waves

all: $(OUT)

$(OUT): $(SRC)
	$(CXX) $(CXXFLAGS) -o $(OUT) $(SRC) $(LDFLAGS)

clean:
	rm -f $(OUT)

.PHONY: all clean
```

> **Внимание**: Файл `glad.c` должен присутствовать в директории сборки (он генерируется утилитой glad и содержит реализации загрузки OpenGL). В приведённом листинге код `glad.c` не показан, но подразумевается его наличие.

### 3.3 Сборка и запуск

```bash
make
./water_waves
```

## 4. Архитектура приложения

### 4.1 Общая структура

Код организован в одном файле `main.cpp` и включает:

- Глобальные константы (разрешение экрана, размер сетки воды, параметры движения кораблей);
- Глобальные переменные состояния (позиция камеры, углы Эйлера, тайминг, флаги переключения);
- Вспомогательные функции загрузки текстур и кубических карт;
- Классы:
  - `Shader` – обёртка для компиляции и линковки GLSL-программ;
  - `Model` – загрузчик мешей через Assimp и их отрисовка;
  - `WaterMesh` – генерация плоской сетки воды (вершины x,z);
- Шейдеры (GLSL), встроенные в исходный код в виде строковых констант;
- Основной цикл рендеринга;
- Callback‑функции ввода.

### 4.2 Класс `Shader`

Находится в строках ~91–167. Обеспечивает:

- Компиляцию вершинного, фрагментного и опционально геометрического шейдера;
- Проверку ошибок компиляции/линковки;
- Удобные методы `use()`, `setMat4()`, `setVec3()`, `setFloat()`, `setInt()`.

### 4.3 Класс `Model` и вложенный `Mesh`

Строки ~170–300. Использует **Assimp** для загрузки модели корабля. Особенности:

- Поддерживаются только диффузные текстуры (тип `aiTextureType_DIFFUSE`);
- Текстуры кешируются в `std::map<std::string, unsigned int> texturesLoaded`;
- Меш содержит собственный VAO, VBO, EBO, количество индексов и ID текстуры;
- Метод `draw()` привязывает текстуру и вызывает `glDrawElements`.

### 4.4 Класс `WaterMesh`

Строки ~303–346. Генерирует плоскую сетку вершин по шаблону:

- Размер сетки: `WATER_GRID_SIZE = 512` вершин в каждом направлении (всего 512×512 вершин);
- Шаг между вершинами: `WATER_SCALE = 0.5` (единицы в мировых координатах);
- Создаются два буфера: вершины (x,z) и индексы для треугольников;
- Метод `draw()` просто отрисовывает индексированные треугольники.

### 4.5 Функции загрузки текстур

- `loadTexture(const char* path)` – загрузка 2D-текстуры (нормаль-карта воды);
- `loadCubemap(std::vector<std::string> faces)` – создание кубической карты из 6 изображений с отражением по горизонтали для корректной ориентации;
- `loadSkybox(bool useSkybox2)` – формирует пути к файлам в зависимости от выбранного набора (`skybox` или `skybox2`) и вызывает `loadCubemap`.

## 5. Шейдеры (GLSL)

### 5.1 Вершинный шейдер воды (`waterVertexShader`)

- Вход: `aPos` – координаты x,z на плоскости.
- Выход: `vTexCoords` – UV-координаты для последующей выборки нормаль-карты (масштаб 0.05).
- Устанавливает `gl_Position` с y=0, передавая управление геометрическому шейдеру.

### 5.2 Геометрический шейдер воды (`waterGeometryShader`)

**Главный вычислительный модуль** волн.

- Вход: треугольники (примитивы) из вершинного шейдера.
- Выход: до 3 вершин с заполненными атрибутами `FragPos`, `Normal`, `TexCoords`, `Height`, `FoamFactor`.
- **Массив волн** `waves[8]`:
  - Каждая волна имеет направление (`dir`), амплитуду (`amplitude`), длину волны (`wavelength`) и крутизну (`steepness`).
  - Параметры подобраны для реалистичного сочетания.
- Функция `displace(p)`:
  - Вычисляет смещение вершины по вертикали и горизонтали (эффект Герстнера с параметром `steepness`).
  - Учитывает глобальный коэффициент `uCalmness`, который уменьшает амплитуду при выборе второго неба (`skybox2`).
  - Циклическая частота вычисляется по формуле дисперсии для глубокой воды: `ω = sqrt(9.81 * 2π / L)`.
- **Нормаль** оценивается конечными разностями (смещение по X и Z на `eps=0.1`).
- **Параметр пены** (`FoamFactor`): `smoothstep(0.3, 0.8, worldPos.y)` – чем выше гребень волны, тем больше пены.

### 5.3 Фрагментный шейдер воды (`waterFragmentShader`)

- Входные атрибуты от геометрического шейдера.
- **Микро-рябь**: выборка из `normalMap` с анимированными UV (`TexCoords * 10.0 + uTime*0.05`) и возмущение основного нормаля.
- Направление отражения: `reflect(-viewDir, perturbedNormal)`.
- **Выборка из кубической карты** `skybox` для получения цвета отражения.
- **Френель** по Шлику:
  - Реалистичный режим (`uFresnelMode == 0`): `R0 = 0.02` (показатель преломления воды ≈1.33), экспонента 5;
  - Усиленный режим (`uFresnelMode == 1`): `R0 = 0.7`, экспонента 2 – очень яркие отражения, почти зеркальные.
- Смешивание цвета воды `(0.0, 0.3, 0.5)` с отражённым цветом по Френелю.
- Наложение пены: смешивание с белым цветом с коэффициентом `FoamFactor * 0.8`.
- Итоговый альфа‑канал 0.9 – лёгкая полупрозрачность.

### 5.4 Шейдеры скайбокса

- Вершинный: передаёт координаты вершины куба, модифицирует `gl_Position = projection * view * vec4(aPos, 1.0);`, затем `pos.xyww` – обнуляет глубину.
- Фрагментный: простая выборка из `skybox` по интерполированным текстурным координатам.

### 5.5 Шейдеры корабля

- Вершинный: преобразует вершины модели в мировое пространство, вычисляет нормали и UV.
- Фрагментный: простой diffuse + ambient с одним направленным источником света (`lightPos`). Текстура модели привязывается как `texture_diffuse1`.

## 6. Модель водной поверхности

### 6.1 Генерация волн

Волны Герстнера реализованы через суперпозицию 8 направленных компонент. Формула для одной волны:

```
вертикаль: y += A * cos(φ)
горизонт: dx -= Q * A * D.x * sin(φ)
          dz -= Q * A * D.y * sin(φ)
φ = (2π/λ) * (D·(x,z)) - ω*t
ω = sqrt(g * 2π/λ), g = 9.81
```

Параметры `waves[8]` заданы в коде как:

| Направление (D)    | Ампл. (A) | Длина волны (λ) | Крутизна (Q) |
|--------------------|-----------|----------------|--------------|
| (1.0, 0.0)         | 0.4       | 10.0           | 0.7          |
| (0.9, 0.1)         | 0.3       | 8.0            | 0.65         |
| (0.95, -0.05)      | 0.25      | 7.0            | 0.6          |
| (0.85, 0.15)       | 0.2       | 11.0           | 0.5          |
| (-0.8, 0.3)        | 0.2       | 9.0            | 0.55         |
| (0.7, 0.2)         | 0.15      | 6.5            | 0.5          |
| (-0.9, 0.05)       | 0.15      | 10.5           | 0.4          |
| (0.6, -0.3)        | 0.1       | 6.0            | 0.4          |

> **Примечание**: `uCalmness` умножается на `A` и `Q`, что позволяет ослабить волны при переключении на спокойное небо (`skybox2`).

### 6.2 Параметры сетки

- Размер сетки: 512×512 вершин → ~262 тыс. вершин, ~523 тыс. треугольников.
- Шаг `WATER_SCALE = 0.5` → полный размер сцены по X и Z: от -128 до +128 (при центре в 0).
- Геометрический шейдер выполняет вычисления для каждой вершины каждого кадра – нагрузка на GPU, но даёт высокую детализацию волн без постоянного обновления буфера вершин.

### 6.3 Рендеринг пены

Пена формируется только на основе высоты вершины (гребни волн). Более реалистичные реализации также учитывали бы кривизну поверхности или скорость, но в данном проекте используется упрощённый метод, дающий приемлемый визуальный эффект.

## 7. Движение кораблей

Два корабля движутся по окружностям разного радиуса (`SHIP_RADIUS = 30.0`) с одинаковой угловой скоростью (`SHIP_ANGULAR_SPEED = 0.15 рад/с`), но с противоположными начальными углами (0 и π). Для каждого корабля:

- Вычисляется текущий угол: `angle = initAngle + angularSpeed * currentTime`.
- Позиция: `(R * cos(angle), 0, R * sin(angle))`.
- Направление касательной: `(-sin(angle), 0, cos(angle))` – используется для поворота модели (только рыскание).
- Модель масштабируется на `SHIP_SCALE = 0.01`.
- Отрисовка выполняется через `drawShip()`, которая строит матрицу `model` с помощью кватернионов.

> **Обратите внимание**: Y‑координата кораблей жёстко задана 0, тогда как вода динамически изменяет высоту. Это приводит к тому, что корабли не «погружены» в волны – упрощение, допустимое для демонстрации.

## 8. Управление и обратные вызовы

### 8.1 Клавиатура

- **W, A, S, D** – перемещение камеры в горизонтальной плоскости (без изменения высоты).
- **B** – переключение между двумя наборами кубических карт (`skybox` ↔ `skybox2`). При переключении сбрасывается флаг `skyboxChanged`, что вызывает перезагрузку текстур в основном цикле.
- **F** – переключение режима Френеля (`realistic` ↔ `enhanced`). В шейдер воды передаётся uniform `uFresnelMode`.
- **ESC** – закрытие окна.

### 8.2 Мышь

- Режим `GLFW_CURSOR_DISABLED` – мышь скрыта, а её движение управляет углами `yaw` и `pitch` камеры.
- Чувствительность: 0.1.
- Ограничение pitch: ±89°.

### 8.3 Изменение размера окна

- Функция `framebuffer_size_callback` обновляет `glViewport`.

## 9. Основной цикл рендеринга

1. Вычисление `deltaTime` и обновление камеры.
2. При необходимости перезагрузка текстур кубической карты.
3. Очистка буферов цвета и глубины.
4. **Рендеринг воды**:
   - Активация `waterShader`, установка uniform-переменных (`uTime`, `uCalmness`, `uFresnelMode`, матрицы, `viewPos`).
   - Привязка нормаль-карты (`GL_TEXTURE0`) и кубической карты (`GL_TEXTURE1`).
   - Вызов `water.draw()`.
5. **Рендеринг скайбокса**:
   - `glDepthFunc(GL_LEQUAL)`, шейдер скайбокса, удаление трансляции из `view`.
   - Отрисовка куба (36 вершин) с привязанной кубической картой.
   - Возврат `glDepthFunc(GL_LESS)`.
6. **Рендеринг кораблей**:
   - Для каждого корабля вызов `drawShip()` с обновлёнными позициями и поворотами.
   - Shader корабля получает `lightPos`, матрицы `model`, `view`, `projection`.
7. `glfwSwapBuffers`, `glfwPollEvents`.

## 10. Особенности реализации

- **Геометрический шейдер воды** выполняет основную вычислительную работу, что позволяет легко менять параметры волн без перегенерации вершинного буфера. Однако на слабых GPU может наблюдаться падение производительности из‑за 512×512 вершин × 8 волн × кадр.
- **Переключение наборов неба** также изменяет `uCalmness` для воды: при спокойном небе (`skybox2`) волны становятся менее выраженными (коэффициент 0.4 против 1.0). Это создаёт визуальную связь между состоянием окружения и динамикой воды.
- **Френель** позволяет наглядно продемонстрировать физический эффект отражения от воды. Усиленный режим полезен для художественных целей.
- **Корабли** используют одну и ту же модель (`gabarre.obj`) с разными матрицами трансформации. Модель должна быть заранее помещена в `models/` вместе со своей текстурой.

## 11. Потенциальные улучшения

- Вычисление высоты волн в вершинном шейдере без геометрического шейдера для повышения производительности.
- Физическое взаимодействие кораблей с водой (определение высоты и наклона).
- Добавление динамических источников света (например, солнца, меняющего направление).
- Реализация каустик и подводного рассеяния.
- Использование текстурных карт для более детализированной пены (белая пена на гребнях с анимацией).

## 12. Заключение

Проект демонстрирует современный подход к рендерингу реалистичной водной поверхности в реальном времени с использованием геометрического шейдера, кубических карт отражений и динамического управления параметрами через пользовательский ввод. Код написан понятно, с разделением на логические блоки, что облегчает его изучение и модификацию.
